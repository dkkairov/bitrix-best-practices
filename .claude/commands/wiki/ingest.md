Выполни операцию **ingest** по схеме из `CLAUDE.md`. Источник: `$ARGUMENTS`

Контент пиши на русском, имена файлов — латиница `kebab-case`. Следуй шаблонам из `wiki/_templates/`.

## Шаги

1. **Определи и сохрани источник** в `raw/sources/`:
   - URL → загрузи (WebFetch), сохрани markdown-снимок как `raw/sources/YYYY-MM-DD-<slug>.md`.
   - Путь к файлу → используй как есть (не дублируй, сошлись на путь).
   - Вставленный текст → сохрани в `raw/sources/YYYY-MM-DD-<slug>.md`.
   `raw/` неизменяем — после сохранения не правим.

2. **Прочитай источник целиком.** Кратко изложи пользователю ключевые выводы и спроси, на чём
   сделать акцент (если неоднозначно).

3. **Создай конспект** в `wiki/sources/` по шаблону `source-summary`: присвой `id` = `S-SS<NN>`
   (NN = следующий по порядку), имя файла `s-ss<NN>-src-<slug>.md`, в frontmatter `aliases: ["src-<slug>"]`
   (TL;DR, тезисы, что встроено, противоречия, открытые вопросы).

4. **Найди 10–15 связанных страниц** (через `index.md`, хабы `_index.md`, Grep по `wiki/`).
   Обнови/создай их: добавь факты, проставь `[[ссылки]]` в обе стороны, обнови `updated`.
   Новые страницы создавай по схеме нумерации (`CLAUDE.md` §4.5): определи раздел по целевой папке и
   `type` → присвой `id` = `<РАЗДЕЛ>-<ТИП><NN>` (NN = max для пары раздел×тип, +1) → положи в нужную
   подпапку (`guides/patterns/antipatterns/reference/`; в cross-cutting/glossary/sources — по их
   правилам) → имя файла `<id-в-нижнем-регистре>-<слаг>.md` → проставь `id` и `aliases` во frontmatter.
   Обновлять `index.md`: добавь строку в «По ID» и инкрементируй счётчик в «Реестре разделов».

5. **REST/события — обязательно сверь** через MCP Bitrix24 (`bitrix-search`,
   `bitrix-method-details`, `bitrix-event-details`). Цитируй `apidocs.bitrix24.ru`. Не выдумывай
   имена методов/полей. Помечай `provenance: documented` со ссылкой.

6. **Противоречия** с существующими страницами не затирай молча: обнови с пометкой что/когда
   изменилось; устаревшее → `status: deprecated` со ссылкой на замену (см. `CLAUDE.md` §6).

7. **Обнови `index.md`** — добавь новые/изменённые страницы в представления.

8. **Допиши `log.md`** (сверху): дата, `ingest`, источник, список затронутых страниц.

9. **Покажи итог** пользователю (что создано/обновлено) и предложи просмотреть в Obsidian.

## Особый случай: источник-каталог (список ссылок)
Если источник — это каталог/список ссылок (awesome-list, индекс документации, портал справки), не
читай всё разом. Заведи его под-источники в `wiki/sources/sources-backlog.md`: классифицируй
(каталог/содержание), привяжи к модулям таксономии, расставь приоритеты — и ингестируй их по одному
позже. REST-методы бери через MCP (`apidocs.bitrix24.ru`), а не скрейпингом `api_help` (старое ядро).
