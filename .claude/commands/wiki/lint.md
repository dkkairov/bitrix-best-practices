Выполни операцию **lint** по схеме из `CLAUDE.md`. Область: `$ARGUMENTS` (пусто = вся вики).

Это диагностика — **не правь автоматически**. Собери отчёт и дай пользователю приоритизировать.
Исключай из проверок `wiki/_templates/` (это шаблоны, не контент).

## Проверки

1. **Битые ссылки:** каждая `[[слаг]]` (без учёта `|подписи`) должна резолвиться в существующий файл
   `wiki/**` — по имени файла **или** по `aliases` в его frontmatter. (Собери все `[[...]]` через
   Grep, сверь с именами файлов через Glob и со списком `aliases`.)
2. **Орфаны:** страницы без входящих ссылок (исключая хабы `_index-*` и шаблоны). У каждой
   контент-страницы должна быть ≥ 1 входящая ссылка.
3. **Устаревшее:** `status: verified` с датой `verified` старше ~6 мес. → пометить на перепроверку.
   `status: deprecated`, на который всё ещё ссылаются как на актуальный.
4. **Ошибки edition:** страницы в `wiki/development/**`, помеченные `cloud`; явно облачные практики,
   помеченные `box`; несоответствие `edition` тексту.
5. **Полнота frontmatter:** обязательные поля на месте; при `status: verified` заполнено `verified`.
6. **Упомянуто, но нет страницы:** концепты/сущности (методы, события, термины) упоминаются, но
   страницы в `glossary/` нет — кандидат на создание.
7. **REST без сверки:** страницы с именами REST-методов/событий, у которых `provenance` ≠
   `documented` или нет ссылки на офф. доку — пометить на сверку через MCP.
8. **Пробелы покрытия:** модули, чьи `_index-<папка>.md` содержат мало/нет страниц.
9. **Дубликаты:** похожие заголовки/темы — кандидаты на объединение.
10. **Имена файлов (`CLAUDE.md` §4.1):** имя контент-страницы — `<тип>-<слаг>.md`, где тип = первое
    слово `type` (`checklist-`/`recipe-`/`pattern-`/`antipattern-`/`concept-`/`entity-`/`source-`);
    префикс соответствует `type`; имя **глобально уникально**. Хабы `_index-*`, шаблоны и
    `sources-backlog` — без типа-префикса (это норма).
11. **Плоская структура:** страницы лежат плоско в папке модуля/области (нет подпапок по типам);
    нет пустых папок/хабов, заведённых заранее.
12. **aliases:** при переименовании прежний слаг сохранён в `aliases` (иначе старые ссылки повиснут);
    `aliases` не дублируются между страницами.

## Отчёт

Сгруппируй находки по серьёзности (🔴 критичное → 🟡 важное → 🔵 улучшения), с указанием файлов и
конкретного действия. В конце предложи топ-5 правок. После согласования можно выполнить безопасные
(добавить обратные ссылки, дозаполнить frontmatter) и записать в `log.md`.
