Выполни операцию **query** по схеме из `CLAUDE.md`. Вопрос: `$ARGUMENTS`

## Шаги

1. **Найди релевантные страницы:** прочитай `index.md`, пройди по хабам `_index.md`, сделай Grep по
   `wiki/` (исключая `wiki/_templates/`).

2. **Прочитай найденные страницы** целиком.

3. **Синтезируй ответ** строго по содержимому вики, со ссылками на страницы-источники в формате
   `[[<id>-слаг|подпись]]`; при необходимости указывай ID решения (напр. «M01-PT01»). Учитывай
   **edition**: если ответ различается для облака и коробки — раздели явно (или уточни, что нужно).

4. **Формат вывода** — по запросу (см. `CLAUDE.md` §7): markdown, таблица сравнения, чек-лист,
   клиентский регламент/методичка, презентация. По умолчанию — markdown.

5. **Помечай надёжность:** если опираешься на `draft`/`empirical`/устаревшее (`verified` давно) —
   предупреди. Если в вики ответа нет — скажи об этом прямо и предложи `/wiki:ingest` источника.

6. **Дофайль находки:** если в ходе ответа выявлено ценное знание, которого нет в вики, предложи
   создать страницу(ы) по шаблонам и, с согласия, создай (с перелинковкой и обновлением `index.md`).

7. **Допиши `log.md`** (одна строка): дата, `query`, суть вопроса, какие страницы использованы/созданы.
