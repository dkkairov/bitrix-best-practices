## Выжимка ключевых правил
- Пространство имён — логическая группировка идентификаторов, чтобы не было конфликтов имён.
- Корень стандартных классов — `Bitrix`; каждый модуль = подпространство по имени модуля:
  `main` → `Bitrix\Main`, `forum` → `Bitrix\Forum`.
- Партнёрские/свои модули используют свой корень, например `Asd\Metrika`.
- Классы модуля располагаются в каталоге `/lib/`.
- Под-пространства (`Bitrix\Main\IO`, `Bitrix\Main\Entity\Validator`) — только при архитектурной
  необходимости.
- Именование: UpperCamelCase, только латиница; имена классов — существительные без лишних аббревиатур.
- `use` для сокращения длинных путей: `use \Bitrix\Main\Localization\Loc;` → `Loc::getMessage('CODE')`.
- Допустимые формы обращения: полный путь `\Bitrix\Main\Class::method()`, сокращённый
  `Main\Class::method()`, алиас через `use`.
