# Чек-лист производительности коробки

**Когда применять:** оптимизация медленного портала на коробке или приёмка нового проекта по
производительности.

## Чек-лист

### 1. Окружение
- [ ] Актуальные версии PHP/MySQL/веб-сервера; включён OPcache
- [ ] Пройден `bitrix_server_test`/проверка системы без критичных замечаний
- [ ] Корректные лимиты PHP (память, время) и настройки веб-сервера

### 2. Кэширование
- [ ] Включён **управляемый кэш** и автокэширование
- [ ] Включён **composite** для публичной части там, где применимо
- [ ] Кэш компонентов настроен; нет отключённого кэша «на всякий случай»
- [ ] Понятна инвалидация кэша по тегам

### 3. База данных
- [ ] Нет медленных запросов в логе; проблемные места проиндексированы
- [ ] Большие выборки — постранично/батчами, без выгрузки «всего сразу»

### 4. Код и фон
- [ ] Нет правок ядра, ломающих кэш/обновления — см. [[box-core-modification]]
- [ ] Агенты вынесены на **cron**, не «на хитах»
- [ ] Тяжёлые операции — в фоне/очередях, не в веб-запросе
- [ ] Картинки и статика оптимизированы, отдаются веб-сервером

### 5. Мониторинг
- [ ] Настроены логи и наблюдение за временем ответа
- [ ] Есть регулярная проверка после релизов

## Критерии приёмки
- Время ответа ключевых страниц в норме под реальной нагрузкой.
- Обновления платформы не ломают кастомизации (нет правок ядра).
