# Роботы и триггеры

**Что это:** no-code инструменты автоматизации CRM, привязанные к стадиям воронки.

## Ключевые факты
| Поле | Значение |
|------|----------|
| Робот | автоматическое действие при достижении стадии (задача, письмо, смена поля/ответственного) |
| Триггер | переводит сделку на стадию в ответ на событие (заявка с формы, звонок, письмо, входящее сообщение) |
| Где работают | лиды, сделки, смарт-процессы ([[g-en01-smart-process|Смарт-процесс (СПА)]]) |
| Edition | cloud и box |

## Как соотносятся
Триггер **запускает** движение по воронке, робот **выполняет** действие на стадии. Вместе образуют
стадийную автоматизацию.

## Подводные камни
- **Порядок роботов** на стадии важен — выполняются последовательно.
- **Зацикливание:** триггер двигает стадию → робот возвращает → снова триггер. Избегать.
- Условия запуска проверять явно, чтобы робот не срабатывал «вхолостую».
- Для сложных маршрутов/согласований — бизнес-процессы ([[m03-pt01-robots-vs-bizproc-decision|Роботы vs бизнес-процессы]]).
