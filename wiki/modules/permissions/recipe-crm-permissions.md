# Рецепт: настройка прав доступа в CRM

**Результат:** менеджеры видят и редактируют только то, что положено по роли; руководители — свои
отделы; данные не «утекают» и не теряются.

## Предусловия
- Построена оргструктура (подразделения и руководители) — она влияет на видимость «по отделу».
- Согласована политика: кто что видит/редактирует/экспортирует.

## Шаги
1. **Определи роли** (например: Менеджер, РОП, Директор, Сервис, Наблюдатель).
2. Для каждой роли задай **уровни доступа** по сущностям (лиды, сделки, контакты, компании, СПА) и
   операциям: чтение, изменение, удаление, **экспорт**, импорт.
3. Используй области видимости: **свои / по отделу / по подразделению (с подчинёнными) / все**.
   Видимость «по отделу» опирается на оргструктуру.
4. **Назначь роли** пользователям/подразделениям.
5. По разным направлениям (`CATEGORY_ID`) можно настраивать права отдельно — используй это для
   разделения процессов (ср. [[checklist-crm-launch|Чек-лист запуска CRM]]).
6. **Протестируй под реальным пользователем** каждой роли (режим «посмотреть как»).

## Проверка результата
- Менеджер не видит чужие сделки (если так задумано); РОП видит свой отдел.
- Экспорт закрыт там, где это критично.
- Новый сотрудник в отделе автоматически получает корректную видимость.

## Откат и проблемы
- «Видно всё» → проверь, не осталась ли роль с областью «все» по умолчанию.
- «Не видно ничего» → пользователь вне нужной роли/подразделения.
- Изменения прав кэшируются — проверяй после полного применения.
